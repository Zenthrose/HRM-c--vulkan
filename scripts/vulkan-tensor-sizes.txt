Vulkan itself does not impose a fixed, universal "tensor size limit" in terms of the total number of elements or bytes. Instead, limits are typically tied to specific resource types and are exposed through the VkPhysicalDeviceProperties structure, which contains various device-specific limits.
Here are some key limits that can impact the effective "tensor size":
Image Dimensions (e.g., maxImageDimension2D, maxImageDimension3D): If a tensor is represented as a Vulkan image (e.g., for textures or compute operations), its dimensions are constrained by these limits. For instance, maxImageDimension2D dictates the maximum width and height for 2D images, and maxImageDimension3D for 3D images. These values are often around 4096 for 2D and 256 for 3D on many devices.
Buffer Ranges (maxTexelBufferElements, maxUniformBufferRange, maxStorageBufferRange): If a tensor is stored in a Vulkan buffer, these limits define the maximum addressable elements in a texel buffer view, or the maximum size of a uniform or storage buffer range that can be bound to a descriptor.
Memory Allocation Size (maxMemoryAllocationSize): This limit specifies the maximum size of a single memory allocation. While a tensor's data might be spread across multiple allocations, a single large tensor might run into this limit if it needs to reside in one contiguous block.
Push Constant Size (maxPushConstantsSize): This limit defines the maximum size of data that can be passed to shaders via push constants, which are a fast, small block of memory. This is typically very small (e.g., 128 or 256 bytes) and not suitable for large tensor data.
Number of Dimensions (Tensor Representation): While not a hard limit in the Vulkan spec itself, some frameworks or libraries built on top of Vulkan might have limitations on the number of dimensions they can efficiently handle for tensors (e.g., a common historical limitation was 4 dimensions due to how metadata was managed in compute shaders).
In summary: The "Vulkan tensor size limit" is not a single, fixed value but a combination of device-specific limits related to image dimensions, buffer sizes, and memory allocation, as well as potential limitations imposed by higher-level frameworks using Vulkan. You need to query the VkPhysicalDeviceProperties of the specific GPU to determine the actual limits for that device.
